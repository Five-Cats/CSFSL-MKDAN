# CSFSL-MKDAN
A Trustworthy Few-Shot Traffic Flow Prediction Model Based on Crowd Sensing

## CSFSL
CSFSL是一个少样本数据集处理方法。经过该方法处理后的数据送入时空序列预测模型STNN后，预测结果较好。

---

* 交通流量数据集NGSIM
* 空气质量数据集AQF

为保证实验设置符合少样本学习场景的研究假设，每个数据集仅保留约3万条时序数据用于实验。

## MKDAN
MKDAN是一个基于跨城市迁移学习的城市交通流量预测模型。它利用多个源城市的数据，通过图神经网络与域自适应技术，对数据稀少的目标城市实现高精度流量预测。

---
MKDAN采用两阶段训练策略：  
**Stage 1 — 对抗预训练（DANN）**  
在多个源城市上训练，利用梯度反转层（GRL）进行全局域对齐，使特征提取器无法区分源域与目标域。  
**Stage 2 — 细粒度分布对齐（MMD微调）**  
在目标城市上微调，通过节点级感知MMD（NodeSim-Aware MMD）进一步缩小源域与目标域的特征分布差距。 
  
核心组件：  
* MVGAT（Multi-View Graph Attention Network）：从邻接矩阵、POI相似度、出发流量、到达流量四种图结构中并行提取空间特征。
* FusionModule：融合多视图特征，采用自注意力 + 交叉注意力 + 门控机制。
* GCN：在时序特征基础上进行逐时间步空间图卷积。
* Temporal Conv：1D卷积捕捉时序依赖。
* DomainAdapter + GRL：节点级域分类器配合梯度反转，实现对抗性域对齐。
* NodeSimAwareMMD：基于节点结构相似度加权的多核MMD损失。

---
时间特征  
模型自动从时间戳中提取以下特征，无需手动准备：  
* 归一化小时（0-1）
* 归一化星期几（day/7,0-1）
* 是否为周末（0/1）

### 实验
step 1：文件data_pre.py即少样本数据集处理方法CSFSL，已采用该方法获取相关处理后数据集。  
step 2：文件main.py为主实验。实验采用随机选取一城市作为目标域，其余三个城市作为源域的设置方式，分别构建四种跨城市迁移场景，评价指标选用MAE和RMSE。

相关代码存放在文件夹others中。

### 消融实验
对比两种配置："none"（无MMD对齐）和 "nodesim"（节点感知MMD），其余所有设置完全相同，控制变量。  
在文件ablation中，四个目标城市（Washington、Chicago、NYCBike、LABike）轮流作为目标，其余三个作为源域；每个目标城市各跑三种 MMD配置（"nodesim", "vanilla", "none"），共12组实验。实验结果保存在文件夹photo中。  

---
### 数据稀疏度实验
在文件data_sparsity_show中，我们分别保留原始训练数据的 50%、30%和 10%作为训练样本，从而模拟不同程度的数据稀缺场景。在每种稀疏度设置下，其余数据保持不变，并重复实验以评估模型在少样本条件下的预测性能。除了我们的模型外，我们根
据前面对比实验中的结果，选取了三个在分别以四个城市为目标城市时效果综合排名靠前的对比模型MTPB、DASTNet、MetaST进行训练。

---
### 参数实验
在每个参数实验中均将对四个城市各自作为目标城市，每个城市下分别用参数5个不同值各独立训练一遍完整的两阶段模型（Stage1 DANN 150 epoch + Stage2 MMD 微调 60 epoch），找到最佳参数。  
实验说明
|     参数     |    取值范围     |   说明   |
|:-----------:|:--------------------------------------:|:---------------------------------|
| mmd_domain |   [0, 0.01, 0.05, 0.10, 0.20]   |   用于控制节点级分布对齐损失在模型训练中的贡献程度，该参数直接影响源域与目标域节点特征分布对齐的强度，mmd_weight=0.10时效果最佳 |
| domain_weight | [0, 0.01, 0.05, 0.10, 0.20] |   用于调节域对抗训练过程中域分类损失的影响程度，从而控制模型在学习任务相关特征与学习域不变特征之间的权衡，domain_weight=0.01时效果最佳 |
| dropout_rate | [0, 0.10, 0.20, 0.30, 0.40] |   则用于缓解模型过拟合问题，通过在训练过程中随机丢弃部分神经元连接，提高模型的泛化能力，dropout_rate=0.30时效果最佳 |
